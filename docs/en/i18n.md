# Internationalization Strategy

🌍 Language:
- 🇬🇧 English (default)
- 🇫🇷 [Français](../fr/i18n.md)

---

## Goal

The project must support English and French labels for all enum values exposed to the frontend.

This is especially important for future UI generation with Lovable.

---

## Core Rule

Enums remain technical and stable.

They are always defined in English.

Translations are handled outside the domain model using stable i18n keys.

---

## Key Format

```text
{EnumName}.{Value}

Example:

GuardrailSeverity.Warning
DocumentStatus.Indexed
RagQueryStatus.BlockedByGuardrail
Why Not Translate Enums Directly?

Enums are part of the technical contract.

They should not contain UI labels or translated values.

Bad example:

public enum GuardrailSeverity
{
    Info,
    Avertissement,
    Bloque
}

Good example:

public enum GuardrailSeverity
{
    Info = 0,
    Warning = 1,
    Blocked = 2
}
API Contract

When enum values are exposed to the frontend, the API should return:

{
  "value": "Warning",
  "code": 1,
  "i18nKey": "GuardrailSeverity.Warning"
}

The frontend can use i18nKey to display the translated label.

MVP Translation Files

Translation files are stored in:

src/SecureRagAgent.Contracts/Localization/en.json
src/SecureRagAgent.Contracts/Localization/fr.json
Design Decision

The Domain layer must not depend on localization.

Localization belongs to the API/UI contract layer.
